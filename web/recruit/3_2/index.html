# 伺服人員招募文件(三期新版文件)

# 招募說明

由於我(服主)未來的生活關係，我的工作時間將會縮短，為了維持伺服的營運，我會逐漸轉往擔任開發與營運的工作。同時，其他現任管理員也有許多私人生活與規劃要處理。

但伺服平時有許多日常事務要處理，在現有的人力編制上將會不足，同時我們也希望有更多新血加入，讓伺服的許多企劃討論能有更多不同的意見與看法。

因此我們將會招募新任的伺服人員，若您有擔任的意願，且認為自己有能力的玩家，歡迎加入我們。

# 工作細項

  • 新玩家的巡視、以及不定期的巡邏
    • 雖然玩家可以檢舉，但新玩家有時候會來破壞與偷竊，或是伺服會有警報提醒通知我們建議去多觀察看看，這時會需要管理員的幫忙。
    • 切記，僅在必要時才可使用此權限，不可用於平常遊玩時的使用。
  • 客服事務處理
    • 進行客服的處理與回覆
    • 如果客服的事情是管理員可以解決的，就幫忙協助處理；而如果是需要服主才能處理的，與玩家溝通與整理
      • 例如: 我東西不見了，與該玩家溝通了解詳細資訊，在了解與整理完成後，請該玩家耐心等待服主處理。簡單說，將客服事務整理成服主能快速理解與處理的資訊。
  • 日常事務處理
    • 商店街事務/美術館事務/換錢事務...
  • 企劃討論參與
    • 我們平常有非常多企劃跟設計需要各位的討論和參與，請多多出席和發表你的想法與意見，這對企劃的發展是很大的幫助。
  • 文件設計與維護
    • 官方網站的資料整理與維護(我們會教你怎麼編輯，但需要你有一些基本的電腦操作能力)。
    • 活動或企劃的文件撰寫與整理
      • 有些企劃需要寫一些文件或表單問券，希望各位能幫上忙分工處理。
      • 有些活動可能會需要文件紀錄的工作(例如有賽程的活動)，希望能協助處理。
  • 平時玩家問題的處理
  • 上述的工作都會教你如何處理，但希望你能主動，我們通常不太會直接指名道姓說誰要來處理這塊，而是大家主動來提出與分工處理。
  • 這些工作雖然一天可能不會花費你多少的時間，但會需要一定的心力與意願。
  • 伺服人員依然為生存玩家，你依然可以像一般玩家自由自在遊玩，僅在處理事情時才可使用相關權限。

# 加入方式

  • 首先，請加入招募群組(寫在Discord公告上)，加入群組後的應徵方式，請看該群組公告
    • 之後將下大格子的問題複製起來，在問題下面 將<你的回答>換成你所要回答的答案
    • 最好能多寫點你的想法
    • 大格子為必填資料，請盡可能詳實撰寫，多多益善。
  • 再來，下面還有一些選填的參考問題，你可以自由挑一些你認為能回答的問題來回答，能讓我們更了解你，也會對應徵上有加分。
  • 接著，我們會與你在該應徵頻道討論，了解雙方的想法
    • 請盡量不要讓我們等你的答案等太久，建議開啟所有通知
    • 該頻道內的討論與你的回答，在尚未經過你的明確同意之前，我們都不會公開。 全程只有現任管理員知道，請放心回答
  • 最後會進行公告招募結果

# 必填問題

  • 基本資料
    • Minecraft ID
      • 請輸入你的遊戲ID
    • Discord ID
      • 必須包含#後面的四個數字
    • 你的年齡
      • 請不要謊報
    • 你目前的事情
      • 如果你目前是學生，請回答你的學位階級(以九月為分界點開始計算 國三/高一/大四...)
      • 如果你不是學生，則請填你目前做的事情(休學/上班族(不用寫職業)/待業...)
    • 平時是否可使用語音?
      • 是/否/可以但不能隨時用...簡略敘述即可
  • 遊戲理解
    • 玩Minecraft多久
      • 你玩這遊戲多久了，回答幾年幾月之類的即可，如果是斷斷續續的玩，則說明一下較頻繁遊玩的期間
    • 目前遊玩本伺服多久
      • 同上
  • 工作時間規劃
    • 若你上任後，你的時間規劃是什麼
      • 上任的時間大約是六月中左右，能簡單說明一下你現實有什麼事情要做嗎?這關係到能否時常上線幫忙的部分，我們也不希望一直要求管理員要現在上線幫忙，進而影響到你的現實生活事務。

# 選填問題說明

  • 下列是選填問題，你可以挑自己喜歡或能回答的問題來回答。
  • 可以都不回答、僅回答部分題目、回答全部問題，對於你認為你特別有想法或這可能能凸顯你的的題目，我們會推薦你多回答看看，能越詳細說明與發表看法越好。
  • 如果你要回答，請自己編排問題與回答方式，好讓我們能一目了然。
  • 每個問題都沒有標準答案，請直接依照自己的想法來分享就好，不要為了想應徵上而回答不符合自己想法的答案。
  • 你也可以自己想一些有想法的題目來主動回答，不一定要限制在下面的問題，總之我們是想盡量了解你。

# 自由選填問題列表

  • 有沒有遊玩過其他的伺服?如果有，能盡可能描述一下該伺服的遊玩狀況嗎?以及你的遊玩心得

  • 有沒有在任何伺服(包括本伺服)擔任過任何伺服人員職位的經驗?

    • 如果有，能否詳細描述一下你的工作內容?
    • 有任何工作上的心得或分享呢?

  • 對於人際溝通與團隊合作參與上，你能舉一些個人經驗或溝通等經驗嗎?

  • 當你認為你在一個團隊中有發生團隊意見與你的意見有不合之地方，你會怎麼面對和處理?

    1. 你認為你的意見對整體計畫比較好(或有其道理在)，而團隊意見對整體計畫比較不好。
    2. 你認為團隊意見對整體計畫比較好(或有其道理在)。
    3. 你認為團隊和你的意見都有一定的好壞與其道理，但你難以接受對方的看法，或是團隊難以接受你的看法。
    4. 你沒有自己的想法，但你覺得團隊意見不好。

  • 若團隊認為人員有一些不適合之處而進行調整和撤換，你會有什麼看法?

  • 對於"管理"這件事，如果你有經驗(不論是遊戲內還生活上)，你的管理心得是什麼?，而如果沒有經驗，你預計會怎面對。

  • 伺服的發展與方向並不全然和每位玩家所想要的一致，而當這狀況發生時，你會怎麼面對和處理。

    • 如果該玩家是伺服的活躍玩家
    • 如果該玩家和你的關係相當密切(同社區的熟人/老朋友)
    • 如果該玩家針對此狀況發表了相當程度的反對與嘲諷

  • 不可避免的，每位伺服人員也是玩家，也會對每位玩家有自己的私人看法與好壞評價，若該玩家你比較或特別不喜歡，在該玩家需要伺服人員的協助時你會怎麼處理。

    • 若該玩家並沒有違反任何規定，但現況對伺服或群體不利時，如果你想要改善，你會怎麼處理。

  • 若你認為你不適合一個團隊時，你會怎麼處理這個狀況。

  • 一間公司的決策考量，應該是優先為了公司著想，還是優先為了客戶著想，還是你有什麼想法?

  • 你會擔心自己的想法無法被團隊所接納，而選擇不發表自己的想法嗎?

    • 或是，你會認為自己的想法是對的，而想盡可能的發表和證明自己嗎?

  • 對於程式與開發上，有什麼經驗可以分享嗎?

  • 對於副本的任務和怪物設計上，有什麼經驗可以分享嗎?

  • 你是個熟悉電腦操作的人嗎?能對這個問題有一些分享嗎?

  • 若突然發生許多急件狀況要同時處理，你會怎麼面對和處理?

  • 你認為你自己在一般玩家之間的形象為何?

  • 若你能成功加入，有什麼特別想做到的事情或想特別幫忙的地方?
